Фільтрація заглушок + статус останньої перевірки у нижній панелі

Проблема/Можливість: У кожній секції vacancies.md перший рядок — - [All vacancies](https://...) — це посилання на сторінку пошуку, а не реальна вакансія. Ці записи потрапляли до UI як картки без оцінки й аналізу, забруднюючи список. Також секція Search URLs парсилась як ще одна секція вакансій. Крім того, нижня панель показувала лише «Підключено» — без жодної інформації про те, коли востаннє запускалась перевірка нових вакансій.

Рішення: 1) У parse_vacancies() (app.py) додано фільтрацію: елементи з назвою All vacancies пропускаються; секції Search URLs та Пошукові запити для самостійного моніторингу ігноруються повністю; порожні секції видаляються з результату. 2) Додано новий ендпоінт GET /api/status, який читає check.log і повертає last_check (ISO timestamp останнього запуску) та added (кількість доданих вакансій із рядка «Done. Added N»). 3) У index.html додано функцію loadStatus(), яка викликається при завантаженні сторінки і вставляє у нижню панель рядок виду «· перевірено 2г тому · +3 нових» або «· нових немає».

Змінені файли: app.py, index.html

Перевірка: venv/bin/python3 -c "import app; print('OK')" — без помилок. parse_vacancies() повертає 4 секції (DOU.ua, Work.ua, NoFluffJobs_looker, NoFluffJobs_bi) без жодних заглушок «All vacancies» та без секції Search URLs. /api/status повертає {"added": null, "last_check": "2026-04-12 03:04:24"}. HTML-валідатор: помилок немає.

Виправлено диспетчер джерел: тепер check_new.py перевіряє всі джерела з config.py

Проблема/Можливість: Функція main() у check_new.py використовувала жорстко закодований список simple_checkers = [check_djinni, check_dou, check_workua], де кожен чекер звертався до фіксованого ключа SOURCES — SOURCES.get("Djinni", ...), SOURCES.get("DOU", ...), SOURCES.get("Work.ua", ...). Але config.py містить окремі джерела Djinni_looker та Djinni_bi (що підтверджується розділами у vacancies.md), які ніколи не перевірялись під час нічного запуску — вакансії з цих пошуків повністю ігнорувались.

Рішення: 1) Рефакторинг check_djinni(), check_dou(), check_workua() — тепер кожна функція приймає source_key як параметр і читає URL та налаштування з SOURCES[source_key], а назву секції в vacancies.md встановлює рівною source_key (замість хардкоду). 2) Перероблено main(): замість статичного списку — єдиний цикл по всіх ключах SOURCES, який диспетчеризує виклик відповідного чекера за доменом URL (djinni → check_djinni, jobs.dou.ua → check_dou, work.ua → check_workua, nofluffjobs → check_nofluffjobs, justjoin → check_justjoin). Тепер усі джерела з config.py автоматично враховуються без зміни коду.

Змінені файли: check_new.py

Перевірка: venv/bin/python3 -c "import check_new; print('OK')" та import app — без помилок. Логіка диспетчера перевірена вручну: кожен ключ SOURCES тепер отримує правильний чекер і додає вакансії до відповідної секції.

2026-04-10 04:00

Undo-скасування видалення + виправлення NameError у check_new.py

Проблема/Можливість: 1) У check_new.py на рядку 578 використовувалась змінна checkers, яка не була визначена у цьому scope — при відсутності нових вакансій це призводило б до NameError під час нічного запуску. 2) Кнопка видалення вакансії викликала нативний браузерний confirm() — незручний модальний діалог без можливості скасування після підтвердження. Один невірний клік — і вакансія безповоротно зникала.

Рішення: 1) Виправлено помилку: замінено len(checkers) на просте повідомлення без undefined-змінної. 2) Замінено confirm() на елегантний undo-снекбар у лівому нижньому куті: після кліку «Видалити» вакансія стає напівпрозорою та зникає, а зникне остаточно лише через 5 секунд. Протягом цих 5 секунд з'являється панель «Вакансію видалено — Скасувати» з анімованою кольоровою смужкою-таймером. Натискання «Скасувати» миттєво відновлює вакансію без звернення до сервера. API-запит на видалення надсилається лише після закінчення 5-секундного вікна.

Змінені файли: check_new.py, index.html

Перевірка: python3 -c "import app; print('OK')" та import check_new — без помилок. HTML-структура валідна. Undo-снекбар містить анімований таймер і кнопку «Скасувати», що коректно скасовує видалення.
